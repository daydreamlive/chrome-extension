# Screenshot Checklist for Store Submissions

This guide explains what screenshots to capture for your Chrome Web Store and Firefox Add-ons submissions.

## Screenshot Requirements

### Chrome Web Store
- **Size**: 1280x800 pixels or 640x400 pixels
- **Format**: PNG or JPEG
- **Count**: Minimum 1, recommended 4-5
- **Maximum**: 5 screenshots

### Firefox Add-ons
- **Size**: Flexible, but 1280x800 recommended for consistency
- **Format**: PNG or JPEG
- **Count**: Recommended 3-5
- **Maximum**: 10 screenshots

---

## Recommended Screenshots

### Screenshot 1: Extension Popup Interface
**Purpose**: Show users the settings interface

**What to capture:**
- Open the extension popup by clicking the extension icon
- Ensure the API key field shows placeholder text (don't show a real API key!)
- Show the prompt textarea with an example prompt like "Apply a subtle beauty filter and enhance colors"
- Make sure the "Save Settings" and "Reset" buttons are visible
- Include the Daydream logo/icon at the top

**Tips:**
- Use a clean browser window
- Capture at 1280x800 resolution
- Consider adding a subtle annotation pointing to key features
- Make sure all text is readable

**Sample caption for store listing:**
```
Configure your AI prompt and API key through the simple settings interface
```

---

### Screenshot 2: Virtual Camera in Video Conferencing App
**Purpose**: Show the virtual camera in action

**What to capture:**
- Open Google Meet, Zoom, or Microsoft Teams
- Open the camera selection dropdown
- Highlight "Chrome Virtual Webcam" in the camera list
- Show the video preview with the AI-processed video

**Tips:**
- Use a recognizable platform (Google Meet is good for Chrome, any for Firefox)
- Make sure the virtual camera name is clearly visible
- Blur or remove any personal information
- Show the video preview if possible

**Sample caption for store listing:**
```
Select "Chrome Virtual Webcam" in any video conferencing app to use AI-enhanced video
```

---

### Screenshot 3: Before/After Comparison
**Purpose**: Demonstrate the AI processing effect

**What to capture:**
- Create a side-by-side comparison image
- Left side: Original webcam feed
- Right side: AI-processed video
- Add labels "Before" and "After"
- Use an obvious but professional AI effect

**Tips:**
- Use good lighting for both captures
- Choose a prompt that shows clear but not exaggerated changes
- Ensure the person in the video has consented to its use
- Make the difference visible but professional

**Sample caption for store listing:**
```
See the difference: AI-enhanced video processing in real-time
```

---

### Screenshot 4: Extension Working in Google Meet
**Purpose**: Show real-world usage

**What to capture:**
- Join a Google Meet call (or use Meet's preview)
- Show the AI-processed video feed
- Include the Meet interface to show it's actually working
- Blur other participants if present

**Tips:**
- Use a test meeting or preview mode
- Show the full Meet interface for context
- Ensure the video quality looks good
- Remove any sensitive meeting information

**Sample caption for store listing:**
```
Works seamlessly with Google Meet, Zoom, Teams, and other platforms
```

---

### Screenshot 5: API Key Setup Flow
**Purpose**: Help users understand the setup process

**What to capture:**
- A composite image showing the setup flow:
  1. Extension icon
  2. Settings popup
  3. app.daydream.live website (blurred in background)
  4. Success message after saving settings

**Tips:**
- Create a visual flow with arrows or numbers
- Show the complete setup process
- Make it look easy and straightforward
- Use annotations if needed

**Sample caption for store listing:**
```
Easy setup: Get your API key from Daydream and configure in seconds
```

## Creating Your Screenshots

### Tools You'll Need
- Chrome or Firefox browser
- Screen capture tool (macOS: Cmd+Shift+4, Windows: Snipping Tool, or use browser extensions)
- Image editor (for resizing, annotations, composites):
  - Free: GIMP, Paint.NET, Photopea
  - Paid: Photoshop, Sketch, Figma

### Step-by-Step Process

1. **Prepare Your Environment**
   - Install the extension in a clean browser profile
   - Clear any personal data
   - Use a test Daydream account
   - Prepare example prompts
   - Have test video conferencing accounts ready

2. **Capture Screenshots**
   - Take screenshots at least 1280x800
   - Capture more than you need (you can select the best ones)
   - Take multiple shots of each scenario
   - Ensure consistent lighting and quality

3. **Edit Screenshots**
   - Resize to exactly 1280x800 pixels
   - Crop to show relevant content
   - Add subtle annotations if needed
   - Blur any personal information
   - Ensure text is readable at all sizes

4. **Add Annotations (Optional)**
   - Use arrows to highlight features
   - Add numbered steps for tutorials
   - Use consistent color scheme (match extension colors)
   - Keep annotations minimal and professional

5. **Optimize Files**
   - Save as PNG for UI screenshots (better quality)
   - Save as JPEG for video content (smaller file size)
   - Compress without losing quality
   - Keep file sizes under 5MB each

## Screenshot Order in Store Listing

**Chrome Web Store recommended order:**
1. Virtual camera in video app (most important)
2. Extension popup interface
3. Before/after comparison
4. Google Meet in action
5. Setup flow

**Firefox Add-ons recommended order:**
Same as Chrome, or adjust based on Firefox-specific features

## Testing Your Screenshots

Before submitting, check:
- [ ] All screenshots are exactly 1280x800 pixels
- [ ] No personal information is visible
- [ ] Text is readable at thumbnail size
- [ ] Colors are consistent across images
- [ ] File sizes are reasonable (<5MB each)
- [ ] Screenshots show the current version's UI
- [ ] No error messages or bugs visible
- [ ] Professional and polished appearance

---

## Alternative Screenshots (If Needed)

If you can't capture certain scenarios, consider:

- **Demo account**: Create a demo Daydream account for screenshots
- **Test environment**: Use local development setup for video processing
- **Mock data**: Use example prompts and placeholder content (clearly marked)
- **Staged demos**: Set up controlled environments for best capture

---

## Captions for Store Listings

Keep captions:
- Under 200 characters
- Descriptive but concise
- Focused on benefits, not features
- Consistent in tone
- Free of jargon

---

## Updating Screenshots

When updating the extension:
- Retake screenshots if UI changes significantly
- Keep screenshot style consistent across updates
- Update at least 1-2 screenshots per major version
- Remove outdated features from screenshots
